# Atelier Codia ユースケースシナリオ

## ステークホルダー

| ステークホルダー | 目的 |
|---|---|
| 訪問者 | 世界観、代表作品、素材感、購入申請の流れを短時間で把握する |
| 購入希望者 | 価格を確認し、購入申請フォームから必要項目を入力する |
| ブランド所有者 | 小さな個人ブランドとして、高級感のある見え方で公開判断する |
| 実装/運用者 | 静的サイトとして安全にビルドし、Pages公開準備まで進める |

## 主要シナリオ

| ID | シナリオ | 成功条件 | 画面 |
|---|---|---|---|
| UC-01 | 初訪問者がトップで世界観を把握する | 大きな写真、短いコピー、控えめなCTAが上質に伝わる | `/` |
| UC-02 | Featured Collectionを見る | 代表作品3〜6点が余白を持って美しく見える | `/` |
| UC-03 | Collectionで価格と作品情報を見る | 作品名、素材、価格、購入申請ボタンが分かる | `/`, `/works` |
| UC-04 | 購入申請を行う | 商品選択、名前、メール、電話、配送先、メモを入力し、疑似完了表示を確認できる | `/request` |
| UC-05 | 購入後の流れを理解する | 後日連絡、銀行振込、発送までの流れが分かる | `/`, `/request` |
| UC-06 | ブランド背景を読む | 手作り、素材へのこだわり、作り手の想いが短く上品に伝わる | `/about` |
| UC-07 | 所有者がローカルで確認し公開判断する | 表示確認・ビルド結果・未決定事項を見て承認可否を判断できる | docs, local preview |
